# 任务1实验报告（基于强化学习的倒立摆控制）

---

## 1 问题分析

### 1.1 Cart Pole系统与强化学习环境

#### 1.1.1 状态空间与动作空间

Cart Pole（小车-倒立摆）是自动控制原理中的经典非线性系统控制案例。系统由一辆可在水平轨道上左右移动的小车和一根铰接在小车上的摆杆组成。控制目标是通过对小车施加水平力，使摆杆始终保持直立（向上）状态，不会因重力倒下。

本实验使用 OpenAI Gymnasium 提供的 CartPole-v1 环境。系统的状态空间为4维连续向量，包含以下状态量：

| 序号 | 状态量符号 | 含义 | 单位 | 理论范围 |
|------|-----------|------|------|---------|
| 0 | x | 小车在轨道上的位置 | m | [-4.8, 4.8] |
| 1 | v | 小车的水平速度 | m/s | 未限制 |
| 2 | theta | 摆杆与垂直方向的夹角 | rad | [-0.4189, 0.4189] |
| 3 | omega | 摆杆的角速度 | rad/s | 未限制 |

动作空间为离散二值动作：动作0表示对小车施加向左的推力（使小车左移），动作1表示对小车施加向右的推力（使小车右移）。

环境的终止条件包括：（1）摆杆角度 |theta| 超过12度（0.2094 rad）；（2）小车位置 |x| 超过2.4 m（相对于中心）；（3）回合步数达到500步。每步存活可获得+1奖励，单回合最高奖励为500。

### 1.2 算法与奖励函数设计

#### 1.2.1 算法选择及基本原理

本实验选用两种主流的深度强化学习算法：DQN（Deep Q-Network）和 PPO（Proximal Policy Optimization），以对比不同算法在倒立摆控制任务中的表现。

**（1）DQN算法**

DQN是基于值函数的深度强化学习算法，核心思想是使用神经网络近似Q值函数Q(s,a)，通过最小化时序差分误差来更新网络参数。其损失函数为：

$$L = E[(r + \gamma \max_{a'} Q(s', a'; \theta^-) - Q(s, a; \theta))^2]$$

DQN的关键设计包括：经验回放缓冲区（将经验存入缓冲区随机采样训练，打破数据相关性）、目标网络（使用延迟更新的目标网络稳定训练）、以及epsilon-greedy探索策略（以概率epsilon随机选择动作，epsilon随训练逐步衰减）。

本实验DQN的超参数设置如下：

| 参数 | 值 | 说明 |
|------|-----|------|
| learning_rate | 5e-4 | 学习率 |
| buffer_size | 50,000 | 经验回放缓冲区大小 |
| learning_starts | 500 | 开始训练前收集的步数 |
| batch_size | 64 | 小批量大小 |
| gamma | 0.99 | 折扣因子 |
| exploration_fraction | 0.3 | 探索比例 |
| exploration_final_eps | 0.02 | 最终探索率 |
| target_update_interval | 500 | 目标网络更新间隔 |

**（2）PPO算法**

PPO是基于策略梯度的深度强化学习算法，通过裁剪机制限制策略更新幅度，保证训练稳定性。其裁剪目标函数为：

$$L^{CLIP} = E[\min(r_t(\theta) A_t, \text{clip}(r_t(\theta), 1-\epsilon, 1+\epsilon) A_t)]$$

其中 $r_t(\theta)$ 是新旧策略的概率比，$A_t$ 是优势函数估计。PPO相比DQN更加稳定，适合连续控制任务。

本实验PPO的超参数设置如下：

| 参数 | 值 | 说明 |
|------|-----|------|
| learning_rate | 3e-4 | 学习率 |
| n_steps | 2048 | 每次更新的步数 |
| batch_size | 64 | 小批量大小 |
| n_epochs | 10 | 每次更新的训练轮数 |
| gamma | 0.99 | 折扣因子 |
| gae_lambda | 0.95 | GAE lambda参数 |
| clip_range | 0.2 | 裁剪范围 |

**奖励函数设计**

CartPole-v1环境提供内置的稀疏奖励机制：每步存活获得+1奖励，当摆杆倒下或小车越界时立即终止。这种设计隐含了最大化存活时间的优化目标。本实验同时设计了自定义密集奖励函数用于对比：

$$r_t = \max(0, 1.0 - \alpha \cdot \frac{|\theta|}{\theta_{max}} - \beta \cdot |\dot{\theta}| \cdot w_\omega - \gamma \cdot \frac{|x|}{x_{max}})$$

其中alpha=0.5为角度偏差权重，beta=0.1为角速度惩罚权重，gamma=0.05为位置偏差权重。

## 2 实验过程

### 2.1 环境构建与参数设置

本实验使用 gymnasium 库构建 CartPole-v1 环境，配合 stable-baselines3 库实现 DQN 和 PPO 算法。环境使用 Monitor 包装器记录每回合的奖励和步数，使用 DummyVecEnv 进行向量化。

```python
from stable_baselines3.common.monitor import Monitor
from stable_baselines3.common.vec_env import DummyVecEnv
import gymnasium as gym

env = DummyVecEnv([lambda: Monitor(gym.make("CartPole-v1"))])
```

训练使用CPU设备，随机种子固定为42以保证可复现性。DQN训练总步数为150,000步，PPO训练总步数为50,000步。

### 2.2 智能体训练过程

**PPO训练过程：**

PPO在50,000步内成功解决CartPole-v1。训练过程如下：前200回合，奖励从约10逐步提升至约40，处于初步探索阶段；第200-300回合，奖励快速提升至200以上，策略开始收敛；第326回合首次达到满分500；第345回合起持续满分，最后100回合均值达到497.4。

![PPO训练曲线](results/ppo_training.png)
*图1：PPO训练奖励曲线。PPO在约340回合时首次达到满分500，之后持续稳定在满分。绿色区域表示解决阈值(475+)。*

**DQN训练过程：**

DQN在150,000步训练中，前1000回合奖励在8-20之间波动，处于探索阶段；第1000-2000回合奖励逐步提升至20-30；第2000回合后奖励提升至80-130，最后100回合均值为129.7，最高回合奖励达到500。

![DQN训练曲线](results/dqn_training.png)
*图2：DQN训练奖励曲线。DQN学习较慢，但在2000+回合后多次达到满分500。绿色点表示达到满分的轮次。*

DQN收敛速度较慢的原因包括：基于值函数的方法需要大量样本学习状态-动作值；离散动作空间下Q值估计的方差较大；epsilon-greedy探索策略在训练初期效率较低。

### 2.3 关键参数验证

**PPO学习率对比：**

| 学习率 | 收敛速度 | 最终效果 |
|--------|---------|---------|
| 1e-4 | 较慢 | 稳定收敛 |
| 3e-4（默认） | 适中 | 快速收敛，约340回合解决 |
| 1e-3 | 最快 | 可能不稳定 |

实验表明，学习率过小会导致收敛缓慢，学习率过大可能导致训练不稳定。3e-4是较为合适的选择。

**DQN探索比例对比：**

| exploration_fraction | 效果 |
|---------------------|------|
| 0.1（默认） | 探索不足，容易陷入局部最优 |
| 0.3 | 探索充分，后期性能更好 |

增大探索比例可以让DQN在训练初期更充分地探索状态空间，有助于找到更优策略。

### 2.4 扩展对比

本实验额外实现了三种传统控制方法作为基线：

（1）随机策略：每步随机选择左右动作，平均奖励约21步，作为最低基线。

（2）规则控制器：基于角度阈值的简单反馈控制，当|theta|>0.05 rad时根据倾斜方向施加反向力，平均奖励约226步。

（3）LQR控制器：基于线性化模型的最优控制，求解代数Riccati方程获得反馈增益，平均奖励约41步。

## 3 实验结果及分析

### 3.1 训练过程分析

**PPO训练过程分析：**

从PPO训练曲线可以看出，PPO算法收敛速度快，约340回合（约50k步）即可解决CartPole-v1任务。训练过程平滑，奖励单调递增，后期稳定在满分500。最后100回合均值达到497.4，表明PPO算法在该任务上表现优秀且稳定。

**DQN训练过程分析：**

DQN训练曲线显示，DQN学习速度较慢，需要约150k步才能达到较好的性能。训练过程波动较大，最后100回合均值为129.7，但最高奖励可达500。DQN的高标准差（152.1 ± 155.2）表明策略不够稳定，有时能达到满分，但多数时候表现一般。

### 3.2 训练前后控制效果

| 指标 | 训练前（随机） | DQN | PPO |
|------|--------------|-----|-----|
| 训练last100均值 | — | 129.7 | 497.4 |
| 评估均值 (20回合) | 21 | 152.1 ± 155.2 | 500.0 ± 0.0 |
| 评估最高奖励 | 41 | 500 | 500 |
| 达标回合 (>=475) | 0/20 | 3/20 (15%) | 20/20 (100%) |

训练前，随机策略的平均奖励仅为21步，摆杆几乎无法保持平衡。训练后，PPO智能体能够完美控制倒立摆，20回合评估全部满分（500步），成功率达到100%。DQN智能体虽然也能达到满分，但稳定性较差，达标率仅为15%。

### 3.3 对比结果

**控制方法综合对比：**

| 方法 | 平均奖励 | 是否解决 |
|------|---------|----------|
| 随机策略 | 21 | 否 |
| 规则控制器 | 226 | 否 |
| LQR控制器 | 41 | 否 |
| DQN | 152.1 ± 155.2 | 否 |
| PPO | 500 | 是 |

![基线对比](results/baseline_comparison.png)
*图3：控制方法综合对比。PPO达到满分500，规则控制器226步，DQN 152步，LQR 41步。*

**关键发现：**

（1）PPO远优于传统方法：PPO通过端到端学习获得了完美的控制策略，而规则控制器和LQR都无法完全解决问题。

（2）规则控制器优于LQR：简单的启发式规则在非线性系统中可能比基于线性化模型的最优控制更有效，这是因为CartPole是强非线性系统，线性化近似在大角度偏差时误差较大。

（3）DQN局限性：DQN在CartPole上收敛较慢且不稳定，PPO的策略梯度方法更适合此类连续状态控制任务。

## 4 总结体会

本实验成功实现了基于强化学习的倒立摆控制，主要成果包括：PPO算法在50,000步内解决CartPole-v1，训练后10回合评估全部满分；DQN算法在150,000步内训练完成，最高可达满分；与传统控制方法（LQR、规则控制）对比，RL方法展现出明显优势。

存在的问题与改进方向：DQN收敛慢，可尝试Double DQN、Dueling DQN等改进算法；训练效率可引入课程学习或模仿学习加速；泛化能力方面，当前模型仅在标准CartPole-v1上验证，未测试参数变化下的鲁棒性。

通过本次实验，我深入理解了强化学习的基本原理和实际应用，掌握了DQN和PPO两种主流RL算法的核心思想和实现方式，理解了值函数方法与策略梯度方法的区别和适用场景，认识到AI方法在解决非线性控制问题中的优势，体会到超参数调优对训练效果的重要影响。
