# 任务1实验报告：基于强化学习的倒立摆控制

---

## 1 问题分析

### 1.1 Cart Pole系统与强化学习环境

#### 1.1.1 状态空间与动作空间

**Cart Pole（小车-倒立摆）** 是自动控制原理中经典的非线性系统控制案例。系统由一辆可在轨道上水平移动的小车和一根铰接在小车上的摆杆组成，控制目标是通过对小车施加水平推力（左移或右移），使摆杆始终保持直立状态。

本实验使用 OpenAI Gymnasium 的 CartPole-v1 物理引擎，并按照任务描述进行了定制：终止角度 15°，失败时施加 -10 惩罚。状态空间为 4 维连续向量：

| 序号 | 符号 | 含义 | 单位 |
|:----:|:----:|------|:----:|
| 0 | x | 小车位置 | m |
| 1 | ẋ | 小车速度 | m/s |
| 2 | θ | 摆杆角度 | rad |
| 3 | θ̇ | 摆杆角速度 | rad/s |

**动作空间：** 离散二值：0 — 左移，1 — 右移。

**终止条件：** （1）摆杆倾斜超过 15°，失去平衡；（2）小车偏离中心超过 2.4 m，滑出轨道；（3）达到 500 步且摆杆未倒下，成功。

### 1.2 算法与奖励函数设计

#### 1.2.1 算法选择及基本原理

**PPO（Proximal Policy Optimization）：** 策略梯度算法，通过裁剪机制（clip_range=0.2）限制更新幅度，稳定性高。超参数：lr=3e-4, n_steps=2048, γ=0.99。

**A2C（Advantage Actor-Critic）：** 演员-评论家算法，经 7 种配置扫描，最优：n_steps=64, lr=5e-4, ent_coef=0.02。

**传统控制方法：** 随机策略、规则控制器、LQR 控制器作为基线。

**奖励函数：** 每步成功 +1，失败 -10。

## 2 实验过程

### 2.1 环境构建与参数设置

使用自定义 CartPole 环境（15°、-10惩罚）。PPO 和 A2C 各训练 200,000 步。随机种子 42，CPU 设备。

| 参数 | PPO | A2C（优化后） |
|:----|:---:|:------------:|
| 学习率 | 3e-4 | 5e-4 |
| n_steps | 2,048 | 64 |
| gamma | 0.99 | 0.99 |
| clip_range | 0.2 | -- |
| ent_coef | 0.0 | 0.02 |

### 2.2 智能体训练过程

**PPO 训练过程：** PPO 在 200,000 步内完美求解（图1）。探索期（0-200回）奖励 10-40；收敛期（200-280回）攀升至 500；求解期（280回后）持续满分，评估 500.0 ± 0.0，100% 达标。

![PPO训练曲线](results_task/fig1_ppo_training.png)

**A2C 训练过程：** A2C 经优化后接近收敛（图2）。缓慢上升期（0-200回）均值 29→227；波动求解期（200-500回）多次满分但波动；稳定提升期（500回后）后100回均值 464.5。评估 493.4 ± 20.8，90% 达标。

![A2C训练曲线](results_task/fig2_a2c_training.png)

### 2.3 关键参数验证

**PPO 学习率对比：** lr=1e-4（后50均值~180）收敛慢，lr=3e-4（~350）最优，lr=1e-3（~280）可能发散。

**A2C 超参数扫描：** n_steps=64, lr=5e-4 最优（后100均值433.5，评估481.9）。n_steps=5 仅41.2，n_steps=512 仅152.3。

### 2.4 扩展对比（传统控制方法）

| 方法 | 是否需要训练 | 训练时间 | 设计复杂度 |
|:----|:-----------:|:--------:|:----------:|
| 随机策略 | 否 | 0 | 无 |
| LQR | 否 | 0 | 高 |
| 规则控制 | 否 | 0 | 低 |
| A2C | 是 | ~3分钟 | 中 |
| PPO | 是 | ~2分钟 | 低 |

## 3 实验结果及分析

### 3.1 训练过程分析

**PPO 分析：** S形曲线，300回后稳定500。滚动均值匀速增长至500后稳定。成功率 0%→60%+（200-300回），稳定在 60%-100%。步数方差从±200缩小至±50。评估 500.0 ± 0.0，100% 达标。

**A2C 分析：** 多尖峰模式，阶梯状增长（150→250→380）。成功率大部分时间接近 0%，后 200 回出现正值不超 30%。后100回均值 464.5，评估 493.4 ± 20.8，90% 达标。

### 3.2 训练前后控制效果

**PPO：** 15分→500分，**31倍**提升，标准差 6.8→0，完全鲁棒。
**A2C：** 15分→493分，**31倍**提升，但有 3 个异常值，标准差 20.8。

![训练前后对比](results_task/fig5_before_after.png)

### 3.3 对比结果

**性能排序：** PPO(500) > A2C(493.4) ≫ 规则(215.8) > LQR(44.8) > 随机(16.1)。PPO 与 A2C 均值差距仅 1.4%，但稳定性差距显著（sd 0 vs 20.8）。

![控制方法对比](results_task/fig3_method_comparison.png)

| 方法 | 平均奖励 | 标准差 | 达标率 | 训练步数 | 是否收敛 |
|:----:|:--------:|:------:|:------:|:---------:|:--------:|
| 随机 | 16.1 | 6.8 | 0% | 0 | 否 |
| LQR | 44.8 | 7.4 | 0% | 0 | 否 |
| 规则 | 215.8 | 29.1 | 0% | 0 | 否 |
| A2C | 493.4 | 20.8 | 90% | 200k | 接近 |
| **PPO** | **500.0** | **0.0** | **100%** | **200k** | **完全** |

## 4 总结体会

**算法效果总结：** PPO 完美解决（500 满分，100%达标）。A2C 评估 493.4，90%达标，接近 PPO。规则控制器（215.8）> LQR（44.8）> 随机（16.1）。

**存在的问题与改进方向：** 算法覆盖度不足、A2C 稳定性不足、超参数搜索不充分、泛化性验证缺失、训练效率可优化。

**个人收获：** 掌握了 PPO 和 A2C 的实现差异，理解了超参数调优的决定性影响，认识到传统控制与 RL 的本质差异。
